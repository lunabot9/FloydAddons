## HARNESS GOTCHAS (cost real time this session ‚Äî heed them)
- **Foreground `sleep` is BLOCKED** ‚Äî a `sleep` in a bash script aborts it (killed a deploy mid-script: client killed, never relaunched). Never use `sleep`; poll with curl-retry loops instead.
- **`prismlauncher --launch` collides** with a running Prism GUI (QLocalSocket). It silently won't start a 2nd copy if an instance is already "running" ‚Äî KILL the old `floyd pvp` java pid first and confirm it's dead, THEN launch. Verify the NEW jar actually loaded via `/state` module names (don't trust that a restart happened).
- **Connecting**: don't re-issue `/action connect` if already connected ‚Äî it disconnects + donutsmp then refuses rapid reconnects ("Failed to connect"). Check `connected` first.
- **Workflow audit/READ agents inherit the session cwd** ‚Äî they audited the WRONG repo (`new-floyd-addons`/`com.odtheking.odin`) when not pinned. ALWAYS pin every agent (impl AND review) to `/Users/twaldin/floyd-pvp-modules` explicitly.

## ANTI-REGRESSION RULES (these exact bugs happened ‚Äî don't reintroduce)
- NEVER call `NVGRenderer`/`RenderSystem`/GL/`nvgCreate` from a class static-init, module init, `ModuleConfig.load`/`Setting.read`, or a `Setting` value-setter ‚Äî runs before the GL context exists ‚Üí `No GLCapabilities` crash on startup (the KeybindSetting boot crash). Defer GL to render time (lazy).
- NEVER add a per-frame reset of a HUD render signal in `Gui.render` ‚Äî it races the HUD pass and blanks the element (the custom-scoreboard regression). The vanilla-scoreboard signal self-clears via `FloydCustomScoreboard.shouldDrawScoreboardHud`'s objective check.
- Block Search match index/render are capped (MAX_INDEXED 100k / MAX_RENDERED 10k nearest) to avoid BufferBuilder vertex overflow + OOM on common blocks.
