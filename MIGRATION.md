# FloydAddons Odin Migration Notes

## Baseline

- Odin source: `odtheking/Odin` `main` at `77b6671`.
- Floyd source: local fork `/Users/twaldin/SkyblockQOLmod` at `17c5ba3`.
- Target Minecraft stack: Fabric 1.21.11, Java 21.

## Architecture Decision

Use Odin's current scaffold as the active application:

- Kotlin module system
- Odin ClickGUI and setting renderers
- Odin config persistence
- Odin event bus and utility layer
- Fabric Loom / official Mojang mappings build

Keep Floyd's game behavior as the implementation source of truth, but port it into Odin modules instead of compiling the Floyd source directly.

The reason is mapping mismatch: Floyd's current source is Java/Yarn (`net.minecraft.client.option`, `net.minecraft.util.Identifier`, etc.), while Odin is Kotlin/official Mojang mappings (`net.minecraft.client.*` Mojang names). Direct copy-in would be a broad mechanical remap plus architecture migration at the same time.

Old Odin gameplay modules have been removed from the active source tree instead of being kept behind Gradle excludes. The source tree now contains the Floyd feature modules, Odin's ClickGUI/module/config/event scaffold, Floyd support helpers, and the vendored Floyd source under `vendor/floydaddons-fabric` for provenance/reference.

The active Odin category model has also been trimmed to Floyd's GUI groups: Render, Hiders, Player, Camera, Cosmetic, QOL, and Misc. Odin's old Dungeon, Boss, Skyblock, and Nether categories are no longer allocated by the active scaffold, and the module package fallback now maps only Floyd package groups.

The active mixin list has been narrowed to Floyd behavior plus Odin scaffolding/event hooks and is packaged as `floydaddons.mixins.json`. Legacy Odin gameplay mixins for `PlayerDisplay`, `HidePlayers`, `RenderOptimizer`, `AutoSprint`, `NoCursorReset`, terminal GUI scaling, teammate highlight colors, Yggdrasil signature bypass, old inventory GUI events, boss-bar suppression, chat-send cancellation, block/entity interaction cancellation, and block-update events are no longer active.

The active startup subscriptions have also been narrowed to Odin scaffolding needed by Floyd: tick scheduling, generic packet/screen/render event dispatch, render batching, GUI widgets, and the Floyd module registry. The event dispatcher now emits client ticks as well as world ticks so Floyd startup/menu features run before a world is loaded. Odin's old SkyBlock, dungeon, Kuudra, party, terminal, scan, server-stat, and split trackers are no longer subscribed at startup.

Floyd setting-group modules that contain their own feature toggles start enabled under Odin so their individual settings behave like the old Floyd GUI toggles instead of requiring a second outer Odin module toggle. The Odin module base also subscribes modules that are constructed with `toggled = true`, so startup-enabled Floyd tick/event hooks actually run before a config file exists. Animations is the exception: Floyd's original `AnimationConfig.enabled` is the behavior gate, so the Odin Animations module starts disabled. Mob ESP remains off by default as a feature toggle, but is `AlwaysActive` so stalk rendering can run independently, matching Floyd's separately registered stalk renderer.

## Ported Behavior

- `FloydRender`: custom world time, server time-packet suppression to prevent flicker, custom window title, and borderless windowed state.
- Custom time conversion now rounds `0..100` slider values to vanilla day ticks like Floyd's `RenderConfig.getCustomTimeTicks()` instead of flooring fractional ticks.
- `FloydAnimations`: first-person held-item position/rotation/scale controls, custom swing duration, swing-restart suppression, cancel re-equip animation, hide empty main hand, and classic-click toggle-key behavior.
- The Animations module now starts disabled by default like Floyd's original `AnimationConfig.enabled` gate; imported legacy `animEnabled` still toggles the Odin module on when a Floyd config had animations enabled.
- `FloydNickHider`: self-name replacement, persisted per-name mapping storage, Hypixel server ID tab/scoreboard tracking, full/abbreviated server ID accumulation, scan-source diagnostics for live tab/scoreboard acquisition, and `/fa name ...` command parity, wired through Odin's existing `FontMixin` text replacement path. The `Server ID Hider` and `Profile ID Hider` GUI toggles live under `FloydHiders`, matching Floyd's v2 Hiders row, while this module owns the replacement implementation.
- Nick Hider's `FormattedCharSequence` replacement path preserves per-character styles for unchanged text and applies the matched text's style to replacement characters, matching Floyd's original `NickTextUtil` behavior instead of flattening changed text to a single style.
- Nick Hider's `Component` replacement path used by `Font.width(FormattedText)` now rebuilds visited component text segments with their original styles instead of flattening the whole component to a single literal when replacements occur. It still falls back to flat replacement for split-across-component regex/date cases, matching Floyd's practical fallback behavior.
- The Nick Hider font hook now also covers Minecraft 1.21.11's `Font.drawInBatch8xOutline(FormattedCharSequence, ...)` outline path, matching Floyd's explicit `drawWithOutline` coverage so outlined nametag/text rendering receives the same replacements as normal prepared text.
- `FloydHiders`: no hurt camera, fire overlay, hunger bar, potion overlay, third-person crosshair, entity fire, attached arrows, falling blocks, explosion particles, tab ping hider, server/profile ID hiders, and no-armor/head-layer hiding for self/others/all.
- `FloydHud`: Odin HUD elements for Floyd's inventory HUD and scoreboard HUD, including movable HUD editor support. Floyd custom scoreboard now suppresses the vanilla sidebar while active, skips live rendering when no real sidebar objective exists, and formats scoreboard lines through the player's scoreboard team prefix/suffix path before drawing.
- `FloydCamera`: freecam, freelook, F5 distance, scroll distance, no-clip, front/back camera skip behavior, freecam attack/use/break suppression, freecam player-input freeze, and disconnect cleanup through Odin settings and Mojang-mapped mixins.
- `FloydXray`: Odin settings, default opaque glass block list, Odin-native opaque-block editor controls, vanilla render-layer forcing, occlusion suppression, full-bright lightmap override, chunk rebuild hooks, Fabric Indigo alpha hook, and optional Sodium alpha/face-culling mixins. Chunk rebuild requests are safe during early config/sidecar load before `levelRenderer` is initialized. Sodium smoke is available through the env-gated Gradle runtime dependency `FLOYDADDONS_SODIUM_RUNTIME=true`.
- `FloydSkin`: custom skin replacement, bundled/default skin extraction, external `config/floydaddons/skins` PNG loading, Odin-native list/reload/previous/next/open-folder controls, and self/others targeting. Fresh Odin defaults now match Floyd's `SkinConfig`: custom skin, self, and others all start disabled, while legacy Floyd config import still maps saved `skinSelfEnabled` / `skinOthersEnabled`.
- `FloydPlayerSize`: Floyd's separate Player Size module entry, with active state derived from XYZ scale, all-axis toggle between `1.0` and `2.0`, per-axis scale controls, and Self/Real Players/All targeting.
- `FloydCape`: Avatar render layer, bundled/default cape extraction, external `config/floydaddons/capes` PNG/GIF loading, Odin-native list/reload/previous/next/open-folder controls, animated GIF frame upload, aspect-aware slab cape rendering, and Floyd cape settings.
- `FloydConeHat`: Avatar render layer, bundled/default cone texture extraction, external `config/floydaddons/cone-hats` PNG loading, Odin-native list/reload/previous/next/open-folder controls, height/radius/y-offset/rotation settings, and textured cone geometry.
- Cosmetic asset fallback now matches Floyd's manager behavior: if the configured skin, cape, or cone image is missing but an asset exists in the corresponding config folder, the first available asset is selected and the Odin config is saved so future launches do not keep pointing at the deleted file.
- `FloydMobEsp`: Odin render-batch tracers and hitboxes, Odin-native filter editor controls, looked-at entity add-name/add-type actions, nearby entity suggestion state, persisted filter maps, per-filter color/chroma metadata, NPC name cache matching, starred armor-stand-to-mob pairing, stable scanned render targets, star-mob miniboss label matching independent of explicit filters, Floyd-style default/per-filter color resolution, player-mob matching for known dungeon minibosses, and runtime stalk tracer support.
